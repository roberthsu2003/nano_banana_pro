### Nano Banana Pro 擅長的 6 大圖解類型

Nano Banana Pro 的核心優勢是**可以在圖片中生成清晰、正確的繁體中文文字**，並理解邏輯順序。

#### 1. 資訊圖表 (Infographics)
這是它最強大的功能。它能將抽象的知識轉化為一張包含標題、數據、插圖和說明文字的長圖。
* **適用場景：** 課程大綱介紹、技術概念懶人包（例如：什麼是 LangChain）。
* **圖片範例描述：** 一張現代風格的長條圖表，背景是淺灰色科技感。標題寫著「Python 學習路徑」。圖表中間有一條蜿蜒的路線，沿途有幾個站點，分別標註著「基礎語法」、「網頁爬蟲」、「資料分析」、「AI 應用」。每個站點旁都有對應的小圖示（如電腦、蜘蛛、圖表）。
* **提示詞建議 (Prompt)：**
    > 請製作一張關於「Python 學習路徑」的資訊圖表。風格要現代簡約。包含四個步驟：1. 基礎語法，2. 網頁爬蟲，3. 資料分析，4. AI 應用。請確保文字清晰可讀。

#### 2. 逐步教學圖解 (Step-by-Step Instructions)
它理解「順序」的概念，可以畫出像 IKEA 說明書或食譜流程的圖。
* **適用場景：** n8n 自動化流程圖、軟體安裝步驟、料理食譜。
* **圖片範例描述：** 一張橫向圖片，分為三個區塊。左邊是「輸入資料」，畫著一個鍵盤和螢幕；中間是「處理」，畫著一個齒輪在轉動；右邊是「輸出結果」，畫著一張報表。箭頭將它們連接起來。
* **提示詞建議 (Prompt)：**
    > 繪製一張「資料處理流程」的三步驟教學圖解。第一步：輸入資料；第二步：處理運算；第三步：輸出結果。風格為平面化設計 (Flat Design)。

#### 3. 技術結構與剖面圖 (Technical & Schematic Illustrations)
它能理解物體的內部構造，畫出類似藍圖或爆炸圖的效果。
* **適用場景：** 解釋 Raspberry Pi 的硬體結構、Docker 的容器概念。
* **圖片範例描述：** 一張 Raspberry Pi 的半寫實插畫，以「爆炸圖」的方式呈現，將電路板、USB 接口、GPIO 針腳分層懸浮展示。旁邊有引線標註著「CPU」、「GPIO」、「USB Port」。
* **提示詞建議 (Prompt)：**
    > 畫一張 Raspberry Pi 開發板的技術結構爆炸圖，詳細展示各個零件層次，風格為精細的工程藍圖，背景為深藍色。

#### 4. UI/UX 原型與介面圖 (UI Mockups)
對於 UI/UX 老師來說這非常實用。它可以生成高保真的 App 或網頁介面圖，且文字不再是亂碼。
* **適用場景：** 展示股票 App 的設計概念、網頁排版範例。
* **圖片範例描述：** 一個智慧型手機的螢幕畫面，顯示著「台灣股市即時通」的 App 介面。頂部是紅色的漲勢大盤曲線，下方有列表顯示「台積電 1000」、「聯發科 1200」。底部有導航列。設計風格類似 iOS 18。
* **提示詞建議 (Prompt)：**
    > 設計一個現代化的股票交易 App 介面圖 (UI Mockup)。標題顯示「台灣股市」，畫面包含即時K線圖和股票列表。配色使用深色模式，強調專業感。

#### 5. 資料視覺化圖表 (Data Visualization)
雖然它不是 Excel，但它可以生成「美學化」的數據圖表，適合用在投影片中吸引目光。
* **適用場景：** 呈現市場趨勢、課程學生分佈。
* **圖片範例描述：** 一個立體的甜甜圈圖 (Donut Chart)，浮在空中。圖表分為三塊，顏色分別是螢光綠、藍色和紫色。旁邊有浮動的標籤寫著「前端 40%」、「後端 30%」、「AI 30%」。材質像玻璃一樣通透。
* **提示詞建議 (Prompt)：**
    > 生成一張 3D 玻璃質感的甜甜圈圖表，展示程式語言的使用比例。標註：Python 50%, JavaScript 30%, 其他 20%。背景要是乾淨的攝影棚燈光。

#### 6. 故事分鏡圖 (Storyboards)
Nano Banana Pro 支援「角色一致性」，可以畫出連環漫畫或分鏡，這對於解釋 User Journey (使用者旅程) 很有用。
* **適用場景：** 描述使用者如何發現問題並使用你的軟體解決問題。
* **圖片範例描述：** 四格漫畫。第一格：一位工程師在電腦前抓頭（表示困惑）；第二格：他發現了 n8n 工具（眼睛一亮）；第三格：他輕鬆地拖拉節點（操作中）；第四格：他喝著咖啡看著自動化完成（滿意）。
* **提示詞建議 (Prompt)：**
    > 製作一個四格分鏡腳本，風格為美式漫畫。描述一位工程師如何使用自動化工具解決工作過勞的問題。角色外觀要保持一致。
